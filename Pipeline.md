## Pipeline:

### Этап 1. Формулировка задачи

Сначала фиксируете, **что именно система должна делать**:

* находить людей на видео;
* проверять наличие СИЗ: каска, жилет, маска;
* выдавать предупреждение, если СИЗ нет;
* при необходимости вести статистику по нарушениям.

Результат этапа: короткое ТЗ на 1–2 страницы.

### Этап 2. Выбор сценария работы

Нужно заранее решить, как будет работать система:

* **видео в реальном времени** с камер;
* **анализ записанных роликов**;
* только **одна камера** или сразу несколько.

Для студенческого проекта лучше начать с:
**одна камера + анализ видео + детекция людей и СИЗ по кадрам**.

### Этап 3. Сбор и подготовка данных

Вам нужны изображения или видео со стройки/похожих объектов.

Что делать:

* собрать датасет;
* выделить классы: `person`, `helmet`, `vest`, `mask`;
* разметить bounding boxes;
* разделить данные на train / val / test.

Результат этапа: готовый размеченный датасет.

### Этап 4. Базовая модель

Сначала делаете простой MVP:

* модель детектирует **людей**;
* затем проверяет, есть ли рядом с человеком **каска / жилет / маска**;
* если объект отсутствует — формируется нарушение.

Обычно для старта берут:

* **YOLOv8 / YOLO11** для детекции;
* простую логику сопоставления: человек + объекты СИЗ в его области.

### Этап 5. Логика проверки

После детекции нужна логика:

* найден человек;
* найдены объекты СИЗ;
* сопоставить СИЗ с конкретным человеком;
* определить статус:

  * всё в порядке;
  * нет каски;
  * нет жилета;
  * нет маски.

Важно: для первого прототипа можно использовать упрощённое правило — считать, что СИЗ находится рядом с человеком в определённой зоне кадра.

### Этап 6. Интерфейс

Сделайте простой интерфейс, где видно:

* видеопоток;
* рамки вокруг людей и СИЗ;
* цветной статус:

  * зелёный — всё нормально;
  * красный — нарушение;
* счётчик нарушений.

Это может быть:

* веб-страница;
* десктоп-программа;
* даже ноутбук-демо в Jupyter + OpenCV.

### Этап 7. Тестирование

Проверяете:

* точность детекции;
* ложные срабатывания;
* работу на разных ракурсах;
* качество при плохом освещении;
* скорость обработки кадров.

### Этап 8. Упаковка проекта

Финальный результат для защиты:

* демонстрационное видео;
* презентация;
* репозиторий с кодом;
* описание архитектуры;
* список ограничений и идей развития.
