# 🧠 ТЗ: ML Engineer #1 — Тренер модели

## Что делаем как команда

Приложение, которое смотрит на фото со стройки и определяет: есть ли на рабочих каска и жилет. Если кого-то нет — показывает красную рамку и пишет "нарушение".

---

## Что ты делаешь простыми словами

Ты учишь компьютер отличать каску от не-каски, жилет от не-жилета. Берёшь готовые фотки, скармливаешь нейронке (она называется YOLO), она учится. На выходе у тебя — файл `best.pt` размером в несколько мегабайт, в котором "записаны мозги" обученной модели. Этот файл ты отдаёшь второму ML-инженеру.

**Ты не пишешь нейронку с нуля.** Используешь готовую библиотеку `ultralytics`, где всё уже сделано. Твоя задача — правильно её запустить и подобрать данные.

### Неделя 3: Обучение первой модели

**Цель:** обучить базовую модель, чтобы вся команда могла начать с ней работать.

1. Загрузи датасет в Google Colab (через **левую панель → Files → Upload**) или подключи Google Drive
2. Запусти обучение:

```python
from ultralytics import YOLO

# Берём предобученную модель и доучиваем под нашу задачу
model = YOLO('yolov8n.pt')  # n = nano, самая лёгкая

results = model.train(
    data='/content/dataset/data.yaml',  # путь к файлу из датасета
    epochs=50,        # 50 эпох — норм для старта
    imgsz=640,        # размер картинок
    batch=16,         # размер пачки (если падает с ошибкой памяти - уменьши до 8)
    name='helmet_v1'  # имя эксперимента
)
```

3. Обучение займёт **1-3 часа**. Не закрывай вкладку Colab! (Если закроешь — обучение прервётся и сессия отвалится.)
4. После окончания файл модели появится в `runs/detect/helmet_v1/weights/best.pt`
5. **Скачай этот файл себе** — это самый важный артефакт

**Если что-то пошло не так:**
- "CUDA out of memory" → уменьши `batch` до 8 или 4
- "No module named ..." → запусти `!pip install ultralytics` ещё раз
- Обучение очень медленное (>5 минут на эпоху) → проверь, что включил GPU в настройках Colab

## 🎯 Ключевые понятия (на случай вопросов от препода)

- **YOLO** — это название архитектуры нейросети для детекции объектов. Расшифровывается "You Only Look Once".
- **Эпоха** — один проход модели по всему датасету. Чем больше эпох, тем лучше учится (до определённого предела).
- **mAP** (mean Average Precision) — главная метрика качества детекции. Чем ближе к 1.0, тем лучше. 0.6+ — приемлемо, 0.8+ — хорошо.
- **Аугментация** — искусственное увеличение датасета: те же картинки, но повёрнутые, с другой яркостью, обрезанные. Помогает модели быть устойчивее.
- **Transfer learning** — то что ты делаешь: берёшь модель, уже обученную на миллионах картинок, и доучиваешь на своих 2000. Это в 100 раз эффективнее, чем учить с нуля.
